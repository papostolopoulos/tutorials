CSS Grid practice

Source

A container element with a display: grid property has to be defined. After that, the following can also be used:

  1. We need to set the column sizes with grid-template-column.
  2. We need to set the row sizes with grid-template-row.
  3. Place the child tags inside the container with grid-column and grid-row properties.

The source order of the child tags does not matter. Similar to flexbox, the CSS grid can place them in any order. This is very convenient when we also want to make order changes based on the different screens that we want the online experience to render at. There is no need for the html to change but the CSS grid can rather help with this change with a few lines of extra code.

Terminology

The terminology that is used in the CSS grid model is important to remember in order to accurately refer to different aspects of the grid structure.

  1. Container: It refers to the tag that contains all the nodes that consitute the grid.
  2. Grid item: This refers to each one of the nodes that constitute the grid (ex: different div tags).
  3. Grid lines: The dividing lines that make the structure of the grid. They can be "column grid lines" or "row grid lines". They reside on either side of a column or a row.
  4. Grid track: The space between two adjacent grid lines. They are like columns or rows of the grid.
  5. Grid cell: The space between two adjacent column grid lines and two adjacent row grid lines. This is similar to a table cell.
  6. Grid area: The total space surrounded by four grid lines. This can be the entire container or part of it.
