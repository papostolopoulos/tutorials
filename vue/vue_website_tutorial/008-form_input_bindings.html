8 - Form Input and Bindings

Basic usage

The v-model directive is a syntactical sugar way for updating data on user input events. This two way data binding directive allows the use of input, textarea and select tags.

v-model ignores the initial value, selected or checked attributes on form elements. It always treats the Vue instance data as the source of truth. For an initial value, it is better if we use the value of the particular data property that is bound to the v-model directive.

Dynamic options rendered with v-for

The following options are binded to an array that is nested in the data of the Vue model. When a different option is selected, the related value is matched to the particular element object of the array, and a different property value from the same element object is displayed.
Dynamic option selection: {{dynamicSelection}}

<select class="" name="" v-model="dynamicSelection"> <option disabled value="" selected>Select an option</option> <option v-for="option in options" :value="option.value">{ {option.option} }</option> </select>

Value bindings

Other than the last example above, all the examples for input - text, input - radio, input - checkbox and select are returning static value attributes. Yet, we can also use v-bind to bind dynamic data from the Vue instance. This option also gives us the opportunity to not just add strings but all sorts of data types.

Example

Checkbox

In the following example, see how the options are changing based on the true-value and false-value attributes added in the input checkbox tag. These attributes do not affect the value of the input checkbox because the browsers do not include unchecked boxes in form submissions. On the other hand, a radio input would guarantee that one of the values is submitted.

Select the checkbox to see how the value changes based on the selection or not of the checkbox: {{toggleCheckbox}}

<input type="checkbox" v-model="toggleCheckbox" id="trueFalseCheckbox" true-value="yes" false-value="no"> <label for="trueFalseCheckbox">Select me</label>
Radio buttons

See how when the checkbox is selected, the v-model inherits the value of the binded attribute.
Radio button v-model: {{radioVModel}}
Radio button value attribute: {{radioBindValue}}

<input type="radio" v-model="radioVModel" v-bind:value="radioBindValue">
Select options

See in the following code how the select options' values are not strings but are objects. They do not necessarily need to be in the data property of the Vue instance but they can be attached directly to the value attribute of the option.

I see you are into {{car.text}}. Would you like to have a {{car.value}}? The car property is: {{car}}.

<select class="" name="" v-model="car"> <option value="Select your car" disabled>Select your favorite cars</option> <option v-for="option in options" :value="option">{ {option.text} }</option> </select>

Modifiers

.lazy

v-model updates the binded data with each event. The .lazy modifier updates the data on the change event only.

Example

This text renders on each event: {{textInput}}

<input type="text" v-model="textInput" name="" value="">

This text renders after the change events. Hit tab or click away from the input tag after you are done entering the text: {{lazyTextInput}}

<input type="text" v-model.lazy="lazyTextInput" name="" value="">
